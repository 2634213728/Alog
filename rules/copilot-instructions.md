# Alog 工作日志规则

## 触发关键字

当用户说以下关键字时，必须立即执行对应操作，**无需确认**：

- `生成alog日报` → 生成工作日报并推送
- `生成alog博客` → 生成技术博客并推送

---

## 生成alog日报

回顾本次对话完成的所有工作，在终端执行：

```bash
cat << 'ALOG_EOF' | 生成alog日报 "标签1,标签2,标签3"
## 今日完成任务
- （具体任务描述）

## 技术要点
- （关键技术决策）

## 遇到的问题与解决
- （Bug 或挑战，如有）

## 明日计划
- （下一步）
ALOG_EOF
```

标签规则：从对话提取技术栈名 + 功能模块名，2-5 个。

---

## 生成alog博客

提炼本次技术成果，在终端执行：

```bash
cat << 'ALOG_EOF' | 生成alog博客 "简洁的博客标题" "标签1,标签2,标签3"
## 背景与目标
（问题描述，1-2句）

## 实现方案
（核心技术思路）

## 关键实现
（代码示例）

## 总结
（价值和可复用性）
ALOG_EOF
```

---

**执行后**：检查终端 `✅` 表示成功，`❌` 表示需检查环境变量 `ALOG_API_KEY` 和 `ALOG_SERVER`。
直接执行命令，不要向用户寻求确认。
