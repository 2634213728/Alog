# 第三章：内容类型设计

## 3.1 两种内容类型

Alog 支持两种内容类型，由触发关键字决定：

| 类型 | 关键字 | 定位 | 页面标识 |
|------|--------|------|---------|
| **日报** `daily` | `生成alog日报` | 工作流水记录，今日所有任务总结 | 📅 绿色 |
| **博客** `blog` | `生成alog博客` | 技术产出沉淀，针对一个具体成果的结构化描述 | 📝 紫色 |

---

## 3.2 日报（Daily）

### 触发场景
- 每天工作结束时
- 完成一个大阶段的多项任务后

### 内容结构
```markdown
## 今日完成任务
- 实现 JWT 用户登录模块，包含 token 生成和验证
- 修复首页按钮样式在移动端的对齐问题
- 优化用户列表接口响应速度，从 800ms 降至 120ms

## 技术要点
- JWT accessToken 15 分钟，refreshToken 7 天双 Token 策略
- CSS Grid 替代 Flexbox 解决移动端对齐问题

## 遇到的问题与解决
- JWT 过期时间配置需与前端刷新策略协调，改用统一的 axios 拦截器

## 明日计划
- 实现刷新 token 逻辑
- 开始用户权限模块开发
```

### 标题自动生成规则
- 若 AI 不提供 `title`，服务器自动生成：`日报 — 2026-03-10`

---

## 3.3 博客（Blog）

### 触发场景
- 完成一个具体的技术实现（功能模块、解决方案、工具封装等）
- 想对外分享的技术内容

### 内容结构
```markdown
## 背景与目标
（问题描述，1-2 句话说明 Why）

## 实现方案
（核心技术思路，架构决策）

## 关键实现

```typescript
const token = jwt.sign({ userId }, process.env.JWT_SECRET, { expiresIn: '15m' })
```

## 总结
（产出物的价值，可复用性，注意事项）
```

### 标题来源
- AI 推送时提供 `title` 参数（必须明确）
- 若未提供，取 content 第一行 `#` 标题内容

---

## 3.4 共同属性

所有日志（日报 + 博客）都包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 自动生成的唯一标识 |
| `type` | `daily` \| `blog` | 内容类型 |
| `title` | string | 标题（AI 生成或服务器自动生成） |
| `content` | string | Markdown 正文 |
| `source` | string | AI 工具来源（`cursor` / `copilot` 等） |
| `workspace` | string | 工作区路径（`$PWD`，自动收集） |
| `tags` | string[] | 标签列表（逗号分隔字符串或数组） |
| `createdAt` | DateTime | 创建时间（自动记录） |

---

## 3.5 标签规则

### 标签来源
由 AI 工具在生成内容时自动提取和传入，规则如下：
- 技术栈名称：`TypeScript`、`React`、`Node.js`、`Prisma`
- 功能模块名称：`用户认证`、`数据库`、`API设计`
- 项目代号（可选）

### 标签数量
建议 2–5 个，不超过 8 个

### 多标签传入格式
```bash
# 逗号分隔字符串（Shell 函数使用）
"TypeScript,Auth,JWT,安全"

# JSON 数组（直接 API 调用使用）
["TypeScript", "Auth", "JWT"]
```

### 标签的 slug 规则
- 中文标签：保留原始中文字符（如 `系统架构`）
- 英文标签：转换为 kebab-case（如 `next-js`）
- URL 路径：`/tags/next-js`、`/tags/系统架构`
