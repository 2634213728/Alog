# 第一章：产品定位与核心需求

## 1.1 产品简介

**Alog（AI Work Log）** 是一个面向 AI 编程工具的任务日志聚合平台。  
本质是一个带有**自动化写入能力**的博客系统，专门收集和展示 AI 辅助编程工作的产出记录。

---

## 1.2 核心使用场景

```
用户使用 AI 工具（Cursor / Copilot / Claude 等）完成编程任务
        ↓
用户对 AI 说：「生成alog日报」或「生成alog博客」
        ↓
AI 自动生成结构化内容，在终端执行推送命令
        ↓
Alog 网站实时展示新日志
```

## 1.3 核心目标

| 目标 | 说明 |
|------|------|
| **零摩擦记录** | 用户只需说关键字，AI 完成所有操作 |
| **多工具聚合** | 支持 Cursor、Copilot、Claude 等多种 AI 工具来源区分 |
| **作者身份** | 每个 API Key 关联一个作者名称，支持同工具区分不同成员 |
| **轻量化部署** | 运行在 Linux 服务器，单进程 + 单文件数据库 |
| **可检索沉淀** | 按标签、来源、类型、作者索引，方便回溯工作历史 |
| **访问摄影** | 每篇日读/博客记录访问次数，了解哪些内容最受关注 |

---

## 1.4 用户角色

**支持多作者模式**。每个 API Key 关联一个作者名称，同一 source（如 copilot）下亦可不同作者登录：

| 来源标识 | 工具 | 与作者关联 | 独立 API Key |
|---------|------|------------|-------------|
| `cursor` | Cursor IDE | 可配置 | ✅ |
| `copilot` | GitHub Copilot | 可配置 | ✅ |
| `claude` | Claude Code | 可配置 | ✅ |
| `windsurf` | Windsurf | 可配置 | ✅ |
| 自定义 | 任意工具 | 可配置 | ✅ |

**示例：** 张三和李四都使用 Copilot，将各自的作者名写入各自的 API Key。推送脏库时页面上就能看到「张三 · copilot」和「李四 · copilot」的内容各自归类。

---

## 1.5 非目标（明确不做）

- ❌ 完整多用户账号系统（无登录/认证）
- ❌ 社交功能（评论、点赞）
- ❌ 实时协作
- ❌ 依赖 Git（不与 git commit 绑定）
- ❌ 需要用户手动打开网站发布

---

## 1.6 MVP 功能范围

### 必须有（Phase 1 + 2）
- [x] 接收日志的 REST API（`POST /api/logs`）
- [x] API Key 管理（区分不同 AI 工具来源）
- [x] 日报 + 博客双内容类型
- [x] 多标签系统，支持按标签筛选
- [x] 日志列表页（全部 / 日报 / 博客）
- [x] 日志详情页（Markdown 渲染）
- [x] Shell 函数客户端（零安装）
- [x] 接入指南页（/setup）

### 待开发（Phase 3）
- [ ] **作者概念**：API Key 关联 `author` 字段，日志自动标注作者
- [ ] **作者页面**：`/authors` 列表所有作者，`/authors/[slug]` 按作者展示其所有日志
- [ ] **访问记录**：每篇日志直接记录浏览次数，卡片和详情页展示访问量

### 后续迭代
- [ ] 全文搜索
- [ ] 工作量统计看板（各工具使用频率、活跃趋势）
- [ ] 导出功能（周报 / 月报）
